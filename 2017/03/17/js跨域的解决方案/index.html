js跨域的解决方案

发表于 2017-03-17 | 分类于 技巧 |

我们在前端开发中经常会遇到跨域请求的问题，下面列举出几种可行的跨域解决方案。

什么是跨域

当我们的项目在开发过程中在一个服务器需要求请求另一个服务器的资源时就会出现跨域访问。这是浏览器为了安全制定了同源策略这个规则，具体情况如下：

  • 不同端口存在跨域：
    http://www.blog.ldaou.com 与 http://www.blog.ldaou.com:8000 他们之间的访问会存在跨域问题。
  • 不同协议存在跨域：
    http://www.blog.ldaou.com 与 https://www.blog.ldaou.com 他们之间的访问会存在跨域。
  • IP与域名
    http://www.blog.ldaou.com 与 http://151.101.100.133 他们也是会存在跨域访问。
  • 同主域不同子域：
    http://www.blog.ldaou.com 与 http://www.ldaou.com 他们之间会存在跨域访问。
  • 不通主域：
    http://www.ldaou.com 与 http://www.iwan0.com 他们之间会存在跨域访问。

其中协议、和端口存在的跨域是需要后台去处理。其中html标签中带有src属性的标签不受同与策略的影响(后面会讲到我们可以利用这一特性做些什么)。

跨域解决方案

一下为几种常见的跨域方案：

  • domain+iframe

  1. 首先建好两个不同域的测试页面，设置他们的domain 为相同的域。

  2. 然后在一个页面 创建iframe

  • 动态创建 script (利用上面讲到的利用script标签的特性 不受同源策略的影响)

  • location.hash + iframe(通过hash来传递)

  1. 在www.a.com/a.html中创建iframe 并使用计时器监听是否变化。

  2. 根据hash值做出反馈(www.b.com/b.html)

  3. 中间页面设置hash值 (www.a.com/c.html)

  • window.name + iframe(利用window.name 的特性)

  1. 在 www.a.com/a.html 设置要传递值

  2. www.a.com/b.html

  • postMessage(利用HTML 5 的 XHR2)

  1. 创建一个www.a.com/a.html 发送数据

  2. 创建一个页面b.html 来接受数据

  • 通过cors实现跨域(服务器配合前端实现)

    服务器通过设置Access-Control-Allow-Origin的值来配合前端实现,当服务器设置值为*是即任何人都可以访问。这样会为网站带来不必要的攻击。

  • jsonp(需要服务端配合)

    jsonp 也是通过javascript 标签不受同源策略的影响特性来实现。

  2. 在www.b.com/b.js 中设置请求
