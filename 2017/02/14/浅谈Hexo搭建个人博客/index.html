浅谈Hexo+Github搭建个人博客

Veröffentlicht am 2017-02-14 | in 教程 |

前言

最近看到许多用Hexo+Github 搭建的个人博客，于是就花了一点时间学习了下，下面是搭建博客的大致步骤。

Hexo 是什么？

Hexo 是一个开源的静态博客生成器, Hexo基于Node.js 使的其变得更加强大、快速、简单。文章编辑为markdown的格式语法，Hexo可以与Github Pages 搭建可访问的博客，较 jekyll， wordpress 要简单方便的多。

Github 为什么可以作为一个类似服务器功能

Github 提供了一个 Github Pages 有300M的免费空间，能自己进行管理，并且使用的git可保证文件可记录

环境要求

安装 Node

Hexo 是基于Node的，其中包管理需要用到npm，生成静态文件也需要依赖Node,进入官网下载安装 Node 下载

安装 Git

Git 是用来将Hexo与Github Pages 联系起来的桥梁，也就是将生成的静态文件上传到Github上，Git 下载

注册Github

Github 最为一个开源的项目托管平台(当然私有是需要收费的) 我们需要用到的是Github上的 托管功能及其Github Pages提供的地址，当然自己如果有域名是最好，传送门注册Github账号

Hexo+Github

上面已经可以在本地进行编写博客和展示，下面使用Github Pages 进行访问。

  1. 首先 需要在github上新建一个仓库，仓库的命名是有规范的 你的github名.github.io(必须)

  2. 回到本地 进入跟目录 有一个_config.yml 文件 修改最底部的 deploy

  3. 使用 hexo deploy命令

  4. hexo deploy 进入github 新建的那个仓库 进入设置选项，也就是Settings 有一个模块为Github Pages 下面有一句 Your site is published at http://blog.ldaou.com/ 点击这个连接即可访问到挂在github Pages上的 博客页面了

Hexo + Github + 绑定个人域名

1.首先需要购买一个域名， 本人在阿里云上购买的域名并将域名备案。
2.进入到阿里云的控制台，点左边的菜单选项 域名与网站(万网) 进入域名解析
3.添加一个域名解析 填写三个选项就可以了。 记录类型：假如你的记录值用的是github的ip那么就用A 用的是Github Pages的引用那就用 CNAME 主机记录： 没有特殊要求就填写@或者www 记录值：填写 github ip 或者是引用 如图：

4.再进入 Github Pages那个模块 将下面那个Custom domain 下的那个输入框填上你的域名 ，点击保存提示保存成功就行了，然后访问自己的域名

总结

Hexo 是一个比较简单好用的博客生成系统，基于node 所带来的快速，便捷和轻量。对于有想法搭建个人博客没地方下手是一个很不错的选择，相比wordpress和jekyll 要方便许多，在搭建过程中虽然也会有很多的坑，当你搭建成功的时候那个坑显然已经不重要了。
