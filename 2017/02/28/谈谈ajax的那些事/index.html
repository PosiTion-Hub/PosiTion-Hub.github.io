谈谈ajax的那些事

发表于 2017-02-28 | 分类于 js |

什么是ajax

AJAX 即“Asynchronous Javascript And XML”（异步JavaScript和XML）,ajax 是一种用于创建快速动态页面的技术，通过ajax可以与后台服务器进行少量的数据交换，ajax可以是页面做到异步更新，也就是说ajax可以做到不重载页面的情况下对某个页面进行部分的更新，在没有出现ajax 页面是需要全部重载实现页面内容的更新。

ajax的工作方式

异步ajax的核心，对于传统的web应用，每次用户想服务器请求数据时，浏览器会丢失当前的页面而从新加载新的数据页面，而原本不需要更新的数据的地方也重新加载了，这就意味着性能的问题，和用户体验问题，比如一个页面你只需要更新table里的一些数据，但是没有ajax 是会重新渲染整个页面，而我们要的仅仅是更新table里面的数据，对于其它的地方的数据我们没有要求更新，所以又浪费了不必要的带宽。当在提交数据到后台获取新的数据是，没有出现ajax 那么页面的渲染是需要这次请求成功后在显示出来 这就意味着数据没有到来的时候，页面其实是空白的，然后就出现了异步请求。

ajax的应用过程：

  1. js脚步创建一个XMLHttpRequest对象。
  2. 使用Request对象的open方法 设置请求路径、是否异步、和请求方式。
  3. 使用request对象设置请求头。
  4. 使用request对象的send方法发送请求，其中send方法可以带请求数据。
  5. 使用Request对象的一个监听事件 去监听请求的状态。

XMLHttpRequest 的属性和方法

XMLHttpRequest 的属性和事件

  • onreadystatechange (监听)
    XMLHttpRequest对象的属性值为一个监听函数，函数可以监听请求的状态。状态变更时触发。
  • readyState (请求状态)
    0：xhr 对象没有初始化
    1：xhr 对象开始发送请求
    2：xhr 对象请求发送完成
    3：xhr 对象读取服务器数据
    4：xhr 对象读取服务器响应数据成功
  • status、statusText (返回状态)
    1.状态码 200: 服务器响应数据成功
    2.状态码 304: 请求服务器资源没有变动，用于缓存
    3.状态吗 400：请求参数错误，服务器无法识别
    4.状态码 403：服务器理解请求，拒绝反应数据
    5.状态吗 404：请求失败，服务器未找到资源
    6.状态码 500：服务器错误
    7.状态吗 504：请求超时
  • responseText (响应数据格式text)
    用于获取服务器的响应文本。
  • responseXML (响应数据格式为xml)
    用于获取服务器响应的XML文档对象。

    XMLHttpRequest 的方法

  • abort()
    停止发送当前请求
  • getAllResponseHeaders()
    获取字符串形式的服务器返回的所有响应头。
  • getResponseHeader(‘headerLabel’)
    依据响应头的名字，获取对应的响应
  • open(‘method’, ‘url’ , asyncFlag)
    建立与服务器url的连接，以及是否使用异步（true/false）
  • send(arguments)
    发送请求，其中content是参数。
  • setRequestHeader(‘label’, ‘value’)
    在发送请求（send）之前，先设置请求头。
